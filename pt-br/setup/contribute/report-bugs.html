Agradecemos seu interesse no Android. Você pode ajudar a melhorar o Android informando problemas e solicitações de recursos no Rastreador de problemas do Android. Esse rastreador contém uma lista de tarefas técnicas pendentes relacionadas a vários tópicos, informações relevantes para essas tarefas e informações sobre o andamento delas, incluindo quais podem ser resolvidas a curto prazo.

Para ver mais informações sobre o motivo pelo qual mudamos para o Rastreador de problemas, consulte esta postagem do blog.

O Rastreador de problemas não é um fórum de suporte ao cliente. Para ver informações de suporte, consulte as Centrais de Ajuda do Nexus e do Pixel. O suporte para outros dispositivos é fornecido pelos fabricantes deles ou pelas operadoras que os vendem.

O suporte para apps do Google ocorre por meio do site de suporte do Google. O suporte para apps de terceiros é fornecido pelo desenvolvedor do app, por exemplo, por meio dos dados de contato fornecidos no Google Play. Para ver uma lista de outros recursos de suporte do Android, consulte nossa página da comunidade.

Não há garantias de que qualquer bug específico será corrigido em uma determinada versão. Para ver o que acontece com seu bug depois que você o informa, leia O ciclo de um bug.

Como registrar um bug

  1. Pesquise seu bug para ver se alguém já o informou. Não se esqueça de pesquisar todos os problemas, não apenas os abertos, porque o problema já pode ter sido informado e fechado. Para encontrar os resultados favoritos, classifique-os pelo número de estrelas.
  2. Se encontrar seu problema e ele for importante para você, marque-o com uma estrela. O número de estrelas em um bug nos ajuda a saber quais correções são mais importantes.
  3. Se ninguém relatou seu bug, informe-o. Primeiro, procure o componente correto, como Framework ou Rede, e preencha o modelo fornecido. Ou selecione a fila de bugs desejada nas tabelas abaixo.

    Dica: alguns componentes contêm subcomponentes, como Rede > Mensagens e Framework > Armazenamento.

  4. Inclua o máximo de informações possível sobre o bug, sempre seguindo as instruções da fila de bugs almejada. Um bug que simplesmente diz que algo não está funcionando não ajuda muito e provavelmente será fechado sem nenhuma ação. A quantidade de detalhes que você fornece, como arquivos de registros, etapas de reprodução e até mesmo um conjunto de patches, ajuda nossa equipe a resolver seu problema.

Filas de bugs

O Rastreador de problemas tem vários subcomponentes em diversas categorias relacionadas ao Android. Há subcomponentes para a segurança, a plataforma, as Ferramentas para Desenvolvedores do Android, a documentação e muito mais.

Segurança

Se você encontrar um problema que afeta a segurança do Android ou de componentes em dispositivos Nexus ou Pixel, siga as instruções indicadas aqui. Além disso, os bugs de segurança estão qualificados para o Programa de recompensas de vulnerabilidades de segurança do Android.

Devido à natureza confidencial dos bugs de segurança, você não poderá procurar problemas abertos, apenas problemas fechados ou problemas que se tornaram públicos.

Ferramentas para Desenvolvedores do Android

Se você encontrar um problema que afeta uma das Ferramentas para Desenvolvedores do Android, como o Android Studio, SDK, Emulador, Imagens do sistema ou Support Library, registre um bug em um desses componentes.

Como as ferramentas têm requisitos diferentes, leia os Detalhes gerais de registro de bugs e os detalhes ligados à ferramenta.
