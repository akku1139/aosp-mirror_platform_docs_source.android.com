Se você quiser ver todos os patches para o Android Open Source Project, ou se estiver revisando ou verificando uma alteração, procure no AOSP Gerrit. Para ver mais informações sobre como encontrar uma alteração específica, consulte Análise de código no Gerrit: como pesquisar alterações (link em inglês).

Como revisar uma alteração

Se você for designado como Revisor de uma alteração, precisará determinar o seguinte:

  • Essa alteração se encaixa no propósito declarado do projeto?
  • Essa alteração é válida dentro da arquitetura existente do projeto?
  • Essa alteração introduz falhas de design que causarão problemas no futuro?
  • Essa alteração segue as práticas recomendadas estabelecidas para o projeto?
  • Essa alteração é uma boa maneira de executar a função descrita?
  • Essa alteração introduz algum risco de segurança ou instabilidade?

Se você aprovar a alteração, marque-a no Gerrit com LGTM (sigla em inglês de "Tudo certo para mim").

Como verificar uma alteração

Se você for designado para ser o Verificador de uma alteração, precisará fazer o seguinte:

  • Aplicar patches da alteração no seu cliente local usando um dos comandos de Download.
  • Criar e testar a alteração.
  • Selecionar o botão Reply (Responder) no Gerrit. Isso abrirá uma caixa de comentários em que é possível marcar a alteração como Verified (Verificada) ou não, além de adicionar uma mensagem explicando quais problemas foram identificados.

Como fazer o download de alterações do Gerrit

Um envio que foi verificado e mesclado será transferido por download com a próxima repo sync. Se você quiser fazer o download de uma alteração específica que ainda não foi aprovada, execute 

repo download TARGET CHANGE

onde TARGET é o diretório local em que a alteração pode ser transferida por download e CHANGE é o número da alteração listado no Gerrit. Para ver mais informações, consulte o artigo Referência de repo.

Como posso me tornar um Verificador ou Revisor?

Em resumo, contribua com código de alta qualidade para um ou mais dos projetos do Android. Para ver mais detalhes sobre as diferentes funções na comunidade do Android Open Source e quem as exerce, consulte Funções do projeto.

Diferenças e comentários

Para abrir os detalhes da alteração no Gerrit, clique no número de identificação ou assunto de uma alteração. Para comparar o código estabelecido com o atualizado, clique no nome do arquivo em Side-by-side diffs (Diferenças lado a lado).

Como adicionar comentários

Qualquer pessoa na comunidade pode usar o Gerrit para adicionar comentários in-line aos envios de código. Um bom comentário será relevante para a linha ou seção do código à qual ele está anexado no Gerrit. Ele pode ser uma sugestão breve e construtiva sobre como uma linha de código pode ser melhorada, ou uma explicação do autor sobre o motivo para o código fazer sentido da maneira como está.

Para adicionar um comentário in-line, clique duas vezes na linha relevante do código e escreva seu comentário na caixa de texto que se abre. Quando você clicar em Save (Salvar), somente você poderá ver o comentário.

Para publicar seus comentários para que outras pessoas que usam o Gerrit possam vê-los, clique no botão "Publish Comments" (Publicar comentários). Seus comentários serão enviados por e-mail para todas as partes relevantes para essa alteração, incluindo o proprietário da alteração, o usuário que fez o envio do patch (caso seja diferente do proprietário) e todos os revisores atuais.
