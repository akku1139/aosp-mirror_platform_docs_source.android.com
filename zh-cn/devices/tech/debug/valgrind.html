Valgrind 工具套件包括 Memcheck（用于检测 C 和 C ++ 中与内存相关的错误）、Cachegrind（缓存分析器）、Massif（堆分析器）和其他几种工具。

注意：尽管您可以使用 Valgrind 对 Android 进行调试，但大多数 Android 平台开发者会改用 AddressSanitizer。

构建 Valgrind

要构建 Valgrind，请运行以下命令：

mmma -j6 external/valgrind

在应用上使用

要在某个应用上使用 Valgrind，请运行以下命令：

adb shell setprop wrap.app_name "logwrapper valgrind"
adb shell am start -a android.intent.action.MAIN -n app_name/.MainActivity
app_name 必须是一个完全符合要求的名称，例如 com.google.example_app。

在系统服务器上使用

要使用 Valgrind 运行系统服务器，请运行以下命令：

adb shell setprop wrap.system_server "logwrapper valgrind"
adb shell stop && adb shell start

获取调试符号

要获取调试符号，请将未剥离的库推送到 /data/local/symbols：

adb shell mkdir /data/local/symbols
adb push $OUT/symbols /data/local/symbols

在启动过程中使用

要在启动过程中使用 Valgrind，请修改 out/target/product/XXXX/root/init.rc，具体方法是将

service example /system/bin/foo --arg1 --arg2

更改为：

>service example /system/bin/logwrapper /system/bin/valgrind /system/bin/foo --arg1 --arg2

要查看效果，请创建一个 boot.img 并重新刷入设备。
